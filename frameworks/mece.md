MECE stands for **Mutually Exclusive, Collectively Exhaustive**.

The MECE Principle is a framework (commonly used in management consulting) for grouping information into categories so analysis is clearer and conclusions are easier to draw.

A MECE categorization follows two rules:

1. **Mutually Exclusive**
   - Each item (or data record) belongs to **one and only one** category.
   - Categories should not overlap.

2. **Collectively Exhaustive**
   - **Every** item (or data record) fits into **at least one** category.
   - The structure should not leave anything out.

**Example:** If you’re categorizing customer support tickets, each ticket should land in exactly one bucket (no overlaps), and all tickets should be captured by the set of buckets (no gaps).
